# 시연 시나리오

## 목적

항만 게이트에서 트랙터/트레일러 번호판을 각각 인식하고, 기사/운송사/작업정보/야드섹터 안내까지 연결되는 흐름을 시연한다.

## 정상 시나리오

| 순서 | 화면 | 동작 | 기대 결과 |
|---|---|---|---|
| 1 | 로그인 | 관리자 계정 로그인 | 관리자 메인 화면 이동 |
| 2 | 번호판 인식 | 트랙터 이미지 업로드 | 트랙터 번호판 인식 성공 |
| 3 | 번호판 인식 | 트랙터 차량 조회 | 기사/운송사 정보 표시 |
| 4 | 번호판 인식 | 트레일러 이미지 업로드 | 트레일러 번호판 인식 성공 |
| 5 | 번호판 인식 | 트레일러 차량 조회 | 작업정보/컨테이너/야드섹터 표시 |
| 6 | 번호판 인식 | 최종 출입 처리 | 통과 처리 및 gate_log 저장 |
| 7 | 기사 화면 | 기사 계정 로그인 | 기사 대시보드 이동 |
| 8 | 기사 화면 | 내 작업 안내 확인 | 본인 작업정보와 야드 안내 확인 |
| 9 | 작업 처리 | 작업 진행 상태 변경 | work_order 상태 변경 |
| 10 | 출차 처리 | 출차 번호판 인식 | 출차 gate_log 저장 |

## 실패 시나리오

| 유형 | 입력 조건 | 기대 결과 | 기록 대상 |
|---|---|---|---|
| 번호판 인식 실패 | 번호판 탐지 실패 이미지 | 출입 불가 메시지 | exception_log |
| 미등록 차량 | DB에 없는 번호판 | 미등록 차량 메시지 | exception_log |
| 차량 유형 불일치 | 트랙터 영역에 트레일러 번호판 | 차량 유형 불일치 메시지 | exception_log |
| 기사 조회 실패 | work_order에 driver_id 없음 | 기사 정보 없음 메시지 | exception_log |
| 기사 출입 불가 | driver.can_enter = false | 기사 출입 불가 메시지 | exception_log |
| 운송사 상태 이상 | carrier_status가 정상 아님 | 운송사 상태 이상 메시지 | exception_log |
| 작업정보 없음 | 트레일러는 등록됐지만 work_order 없음 | 작업정보 없음 메시지 | exception_log |
| 작업 미승인 | work_order.is_approved = false | 작업 미승인 메시지 | exception_log |
| 야드섹터 없음 | container.sector_id 없음 | 야드 안내 불가 메시지 | exception_log |

## 발표 시 강조 포인트

| 포인트 | 설명 |
|---|---|
| AI 핵심성 | YOLO로 번호판 영역을 탐지하고 OCR로 문자열을 추출 |
| 항만 업무 반영 | 트랙터와 트레일러를 분리 인식하여 서로 다른 정보를 조회 |
| 트랙터 조회 | 기사/운송사/출입 가능 여부 확인 |
| 트레일러 조회 | 작업오더/컨테이너/야드섹터 안내 확인 |
| 통합 판단 | 두 번호판 조건이 모두 맞을 때만 출입 가능 |
| 기사 화면 | 기사가 로그인하면 본인 작업과 야드 안내를 확인 |
